### `list`
this command lists all the subscriptions available in your 
environment for a specific topic or source.

**Options:**
<table>
<tr><th>Option</th><th>Description</th></tr>
<tr><td>`source`</td><td>The name of the source.</td></tr>
<tr><td>`topic`</td><td>The name of the topic.</td></tr>
</table>
**Examples:**
```sh
qala subscriptions ls --topic <TOPIC_NAME>
qala subscriptions ls --source  <SOURCE_NAME>
```


### `inspect`
this command retrieves the subscription with the specified 
ID.

**Options:**
<table>
<tr><th>Option</th><th>Description</th></tr>
<tr><td>`source`</td><td>The name of the source.</td></tr>
<tr><td>`s` / `subscription`</td><td>The name of the subscription.</td></tr>
<tr><td>`topic`</td><td>The name of the topic.</td></tr>
</table>
**Examples:**
```sh
qala subscriptions i --topic <TOPIC_NAME> -s <SUBSCRIPTION_NAME>
qala subscriptions i --source <SOURCE_NAME>  -s <SUBSCRIPTION_NAME>
```


### `create`
this command creates a new subscription for the Qala 
CLI.

**Options:**
<table>
<tr><th>Option</th><th>Description</th></tr>
<tr><td>`a` / `audience`</td><td>The audience to scope the subscription - for 
topics.</td></tr>
<tr><td>`d` / `description`</td><td>The description of the subscription.</td></tr>
<tr><td>`e` / `events`</td><td>The comma separated list of event type 
names.</td></tr>
<tr><td>`m` / `max-attempts`</td><td>The maximum delivery attempts of the 
subscription.</td></tr>
<tr><td>`n` / `name`</td><td>The name of the subscription.</td></tr>
<tr><td>`source`</td><td>The name of the source.</td></tr>
<tr><td>`topic`</td><td>The name of the topic.</td></tr>
<tr><td>`u` / `url`</td><td>The webhook url of the subscription.</td></tr>
</table>
**Examples:**
```sh
qala sub create -n <SUBSCRIPTION_NAME>  --topic <TOPIC_NAME> -d <DESCRIPTION> -e  <EVENTS_COMMA_SEPERATED_NAMES> -u <WEBHOOK_URL> -m  <MAX_DELIVERY_ATTEMPTS> -a <AUDIENCE>
```


### `update`
this command updates an existing subscription for the Qala 
CLI.

**Options:**
<table>
<tr><th>Option</th><th>Description</th></tr>
<tr><td>`a` / `audience`</td><td>The audience to scope the subscription - for 
topics.</td></tr>
<tr><td>`d` / `description`</td><td>The description of the subscription.</td></tr>
<tr><td>`e` / `events`</td><td>The comma separated list of event type 
names.</td></tr>
<tr><td>`m` / `max-attempts`</td><td>The maximum delivery attempts of the 
subscription.</td></tr>
<tr><td>`n` / `name`</td><td>The new name of the subscription.</td></tr>
<tr><td>`source`</td><td>The name of the source.</td></tr>
<tr><td>`topic`</td><td>The name of the topic.</td></tr>
<tr><td>`u` / `url`</td><td>The webhook url of the subscription.</td></tr>
</table>
**Examples:**
```sh
qala sub update <SUBSCRIPTION_NAME> --topic  <TOPIC_NAME> -n <NEW_NAME> -d <DESCRIPTION> -e  <EVENTS_COMMA_SEPERATED_NAMES> -u <WEBHOOK_URL> -m  <MAX_DELIVERY_ATTEMPTS> -a <AUDIENCE>
```


### `delete`
this command deletes the subscription with the specified 
ID.

**Options:**
<table>
<tr><th>Option</th><th>Description</th></tr>
<tr><td>`source`</td><td>The name of the source.</td></tr>
<tr><td>`s` / `subscription`</td><td>The name of the subscription.</td></tr>
<tr><td>`topic`</td><td>The name of the topic.</td></tr>
</table>
**Examples:**
```sh
qala sub delete --topic <TOPIC_NAME>  --subscription <SUBSCRIPTION_NAME>
```
